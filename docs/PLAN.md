## 2. 핵심 가치 제안

> **"GitHub 활동 데이터를 기반으로 당신의 개발 스타일을 분석했습니다"**  
> ※ 실제 성격이 아닌, 코딩 패턴 기반 추정입니다

GitHub 활동 패턴(커밋 빈도, 시간대, 언어 분포, 기여 규모)을 지표별 점수로 환산해 개발자의 코딩 성향을 8가지 유형 중 하나로 도출합니다.  
외부 AI API 없이 규칙 기반 스코어링만 사용해 비용 없이 빠르게 결과를 제공하고, 결과를 README 배지로 공유할 수 있어 바이럴 확산이 유리합니다.

### 분석 데이터 소스 (GitHub API)

- 커밋 빈도 · 날짜 분포
- 커밋 크기 (변경 라인 수)
- 활동 시간대
- 사용 언어 수
- 본인 repo 수
- fork 비율
- 활동 연속성
- 커밋 메시지 패턴

## 3. 개발자 성향 유형 (8가지)

### 🌱 꾸준형 (Gardener)

작은 변화를 꾸준히 쌓아가는 개발 스타일. 매일 코드를 만지며 안정적인 리듬으로 프로젝트를 진행합니다.

| 지표      | 신호              |
| --------- | ----------------- |
| 활동 빈도 | 30일 중 18일 이상 |
| 커밋 크기 | 평균 < 50라인     |
| 활동 편차 | 낮음 (안정적)     |
| 주된 작업 | 유지보수 · 소수정 |

---

### 🏃 몰입형 (Sprinter)

평소엔 조용하다가 특정 기간에 폭발적으로 집중하는 스타일. 한 번 몰입하면 대량의 코드를 쏟아냅니다.

| 지표             | 신호                |
| ---------------- | ------------------- |
| 활동 편차        | 매우 높음           |
| 최대 단일일 커밋 | 20커밋 이상         |
| 커밋 크기        | 평균 > 150라인      |
| 패턴             | 폭발적 집중 후 휴식 |

---

### 🧱 설계형 (Architect)

커밋 수는 적지만 한 번 올릴 때 구조적인 변화를 만듭니다. 리팩토링과 설계 작업을 중시합니다.

| 지표        | 신호                   |
| ----------- | ---------------------- |
| 커밋 빈도   | 낮음                   |
| 커밋 크기   | 크고 영향 범위 넓음    |
| 메시지 패턴 | refactor / restructure |
| 주된 작업   | 구조 변경 · 설계       |

---

### ⚡ 실험형 (Hacker)

빠르게 만들고 바로 시도해보는 스타일. 야간에 특히 활발하고 다양한 실험을 즐깁니다.

| 지표      | 신호              |
| --------- | ----------------- |
| 야간 비율 | 30% 이상          |
| 브랜치 수 | 많음 (실험용)     |
| 커밋 속도 | 빠름              |
| 주된 작업 | 프로토타입 · 실험 |

---

### 📚 탐구형 (Researcher)

깊이 있게 탐구하고 분석하는 개발 스타일. 데이터 분석 코드나 실험 노트북이 많습니다.

| 지표      | 신호               |
| --------- | ------------------ |
| 주요 언어 | Python / Jupyter   |
| repo 성격 | 분석 · 실험 코드   |
| 문서화    | README 상세        |
| 주된 작업 | 탐구 · 데이터 분석 |

---

### 🔧 장인형 (Craftsman)

디테일과 완성도를 중요하게 여깁니다. 작은 수정을 반복하고 테스트와 문서화에 공을 들입니다.

| 지표        | 신호                 |
| ----------- | -------------------- |
| 커밋 크기   | 매우 작음 (< 20라인) |
| 메시지 패턴 | fix / chore / docs   |
| 테스트 파일 | 비율 높음            |
| 주된 작업   | 정제 · 품질 개선     |

---

### 🧭 탐험형 (Explorer)

새로운 기술을 끊임없이 시도합니다. 여러 언어와 다양한 프로젝트를 넘나드는 것이 특징입니다.

| 지표        | 신호         |
| ----------- | ------------ |
| 사용 언어   | 5개 이상     |
| fork 비율   | 높음         |
| 프로젝트 수 | 다수 · 다양  |
| 주된 작업   | 새 기술 탐색 |

---

### 🛠️ 빌더형 (Builder)

무언가를 꾸준히 만들어내는 실행형 스타일. 자체 repo가 많고 지속적으로 개발을 이어갑니다.

| 지표        | 신호             |
| ----------- | ---------------- |
| 자체 repo   | 10개 이상        |
| 활동 지속성 | 장기 지속        |
| 커밋 메시지 | feat / add 중심  |
| 주된 작업   | 기능 개발 · 실용 |

## 3.5. 유형별 캐릭터 디자인

각 유형의 코딩 성향을 직관적으로 표현하는 캐릭터를 결과 화면과 SNS 공유 카드에 활용합니다.

### 캐릭터 공통 가이드

| 항목   | 방향                                      |
| ------ | ----------------------------------------- |
| 스타일 | 픽셀 아트                                 |
| 형태   | 개발자 느낌의 캐릭터 (사람형 or 마스코트) |
| 표현   | 포즈 + 소품으로 유형 성향 시각화          |
| 컬러   | 유형별 대표 색상 지정                     |

---

### 🌱 꾸준형 (Gardener)

- **컨셉**: 화분을 정성껏 가꾸는 개발자
- **포즈**: 물뿌리개를 든 채 화분 앞에 앉아 있는 모습
- **소품**: 물뿌리개, 작은 화분들, 달력 (매일 체크 표시)
- **키워드**: 편안함, 꾸준함, 온기
- **대표 색상**: `#4CAF50` (초록)

---

### 🏃 몰입형 (Sprinter)

- **컨셉**: 에너지가 폭발하는 스프린터 개발자
- **포즈**: 키보드에 양손을 올리고 앞으로 몸을 기울인 집중 자세
- **소품**: 에너지드링크 캔, 헤드폰, 화면 가득한 코드
- **키워드**: 긴장감, 폭발적 에너지, 집중
- **대표 색상**: `#FF5722` (주황-레드)

---

### 🧱 설계형 (Architect)

- **컨셉**: 청사진을 펼쳐놓고 구조를 설계하는 개발자
- **포즈**: 큰 설계도 앞에 서서 턱을 괴고 분석하는 모습
- **소품**: 청사진, 자, 연필, 안경
- **키워드**: 신중함, 전략적, 지적
- **대표 색상**: `#5C6BC0` (인디고)

---

### ⚡ 실험형 (Hacker)

- **컨셉**: 깊은 밤 여러 모니터 앞에서 실험하는 개발자
- **포즈**: 어두운 방에서 여러 화면을 바라보며 뭔가를 시도하는 모습
- **소품**: 멀티 모니터, 후드티, 터미널 화면, 번개 이펙트
- **키워드**: 야간, 실험적, 빠름
- **대표 색상**: `#FFC107` (앰버)

---

### 📚 탐구형 (Researcher)

- **컨셉**: 책과 데이터 속에서 탐구하는 개발자
- **포즈**: 책과 논문이 쌓인 책상에서 노트에 뭔가를 적는 모습
- **소품**: 책 더미, 돋보기, 그래프/차트, Jupyter 노트북
- **키워드**: 호기심, 깊이, 학구적
- **대표 색상**: `#26A69A` (틸)

---

### 🔧 장인형 (Craftsman)

- **컨셉**: 정밀 도구로 섬세하게 작업하는 장인 개발자
- **포즈**: 작은 부품을 핀셋으로 정밀하게 다루는 모습
- **소품**: 핀셋, 작업용 돋보기, 정리된 공구함, 체크리스트
- **키워드**: 섬세함, 완성도, 장인 정신
- **대표 색상**: `#8D6E63` (브라운)

- **포즈**: 나침반을 들고 여러 방향을 가리키며 걷는 모습
- **소품**: 나침반, 지도, 모험가 가방, 다양한 언어 로고 깃발
- **키워드**: 호기심, 자유로움, 다양성
- **대표 색상**: `#42A5F5` (하늘색)

- **소품**: 망치, 스패너, 설계도, 만들다 만 프로젝트들
- **키워드**: 실행력, 생산성, 에너지
- **대표 색상**: `#EF5350` (레드)

## 4. 분석 로직

### 스코어링 방식

각 지표를 측정해 유형별 가중 점수를 합산합니다.  
가장 높은 점수의 유형이 메인 결과로 출력됩니다.  
점수 차가 10% 이내면 "혼합형" 표기를 추가합니다 (예: 꾸준형 + 장인형).  
모든 계산은 백엔드 Python 로직으로 처리하며 외부 AI API를 사용하지 않습니다.

### 주요 지표 정의

| 지표        | 측정 방법                                        |
| ----------- | ------------------------------------------------ |
| 활동 연속성 | 최근 30일 중 커밋이 있는 날 수                   |
| 집중도      | 단일 날짜 최대 커밋 수                           |
| 커밋 크기   | 커밋당 평균 변경 라인 수 (additions + deletions) |
| 야간 비율   | 22시~04시 사이 커밋 수 / 전체 커밋 수            |
| 언어 다양성 | repo 전반에서 사용된 언어 종류 수                |

### 꾸준형 판별 예시

```
활동일 > 20일        +30점
평균 커밋 크기 < 50라인  +25점
주말 활동 있음        +15점
야간 비율 < 20%      +10점
```

## 5. 결과 출력 구조

### 유형 궁합

각 유형별 잘 맞는 유형 1개, 안 맞는 유형 1개를 결과 페이지에 표시한다.

```ts
const COMPATIBILITY: Record<string, { good: string; bad: string }> = {
  gardener: { good: "craftsman", bad: "sprinter" }, // 묵묵히 퀄리티 vs 리듬 불일치
  sprinter: { good: "hacker", bad: "researcher" }, // 둘 다 먼저 돌진 vs 원리 파는 동안 배포
  architect: { good: "researcher", bad: "hacker" }, // 깊이 이해 후 행동 vs 다이어그램 없이 누름
  hacker: { good: "explorer", bad: "craftsman" }, // 새 영역 탐험 vs 완벽주의 충돌
  researcher: { good: "architect", bad: "builder" }, // 이해→설계→실행 vs 일단 배포
  craftsman: { good: "architect", bad: "hacker" }, // 구조+디테일 vs 코드 리뷰 전쟁
  explorer: { good: "hacker", bad: "gardener" }, // 신기술 PoC 최강 vs 한 우물 충돌
  builder: { good: "sprinter", bad: "architect" }, // 폭발적 생산성 vs 설계 끝날 때까지 대기
};
```

```

```

### README 배지

```markdown
![dev 8ersonality](https://dev-8ersonality.com/badge/username)
```

출력 예시: `꾸준형 🌱 | Gardener`

### 이미지 저장

Canvas API로 1080×1920 PNG 생성. 유형명 + 스프라이트 + 훅 문구 + 설명 포함. 이미지로 저장하기 버튼으로 로컬 다운로드.

## 6. 사용자 경험 흐름

```
GitHub 로그인 (OAuth)
    ↓
분석 기간 선택 (30 / 90 / 180일)
    ↓
스코어링 분석 (1~2초)
  "커밋 패턴 분석 중..."
  "야간 활동 비율 계산 중..."
  "성향 도출 중..."
    ↓
결과 출력 (유형 + 근거)
    ↓
공유 (이미지 저장 / URL 복사)
```
