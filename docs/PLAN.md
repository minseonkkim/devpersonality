## 2. 핵심 가치 제안

> **"GitHub 활동 데이터를 기반으로 당신의 개발 스타일을 분석했습니다"**  
> ※ 실제 성격이 아닌, 코딩 패턴 기반 추정입니다

GitHub 활동 패턴(커밋 빈도, 시간대, 언어 분포, 기여 규모)을 분석해 개발자의 코딩 성향을 8가지 유형 중 하나로 도출합니다.  
실제 행동 데이터 기반이라 신뢰도가 높고, 결과를 README 배지로 공유할 수 있어 바이럴 확산이 유리합니다.

### 분석 데이터 소스 (GitHub API)

- 커밋 빈도 · 날짜 분포
- 커밋 크기 (변경 라인 수)
- 활동 시간대
- 사용 언어 수
- 본인 repo 수
- fork 비율
- 활동 연속성
- 커밋 메시지 패턴

## 3. 개발자 성향 유형 (8가지)

### 🌱 꾸준형 (Gardener)

작은 변화를 꾸준히 쌓아가는 개발 스타일. 매일 코드를 만지며 안정적인 리듬으로 프로젝트를 진행합니다.

| 지표      | 신호              |
| --------- | ----------------- |
| 활동 빈도 | 30일 중 18일 이상 |
| 커밋 크기 | 평균 < 50라인     |
| 활동 편차 | 낮음 (안정적)     |
| 주된 작업 | 유지보수 · 소수정 |

---

### 🏃 몰입형 (Sprinter)

평소엔 조용하다가 특정 기간에 폭발적으로 집중하는 스타일. 한 번 몰입하면 대량의 코드를 쏟아냅니다.

| 지표             | 신호                |
| ---------------- | ------------------- |
| 활동 편차        | 매우 높음           |
| 최대 단일일 커밋 | 20커밋 이상         |
| 커밋 크기        | 평균 > 150라인      |
| 패턴             | 폭발적 집중 후 휴식 |

---

### 🧱 설계형 (Architect)

커밋 수는 적지만 한 번 올릴 때 구조적인 변화를 만듭니다. 리팩토링과 설계 작업을 중시합니다.

| 지표        | 신호                   |
| ----------- | ---------------------- |
| 커밋 빈도   | 낮음                   |
| 커밋 크기   | 크고 영향 범위 넓음    |
| 메시지 패턴 | refactor / restructure |
| 주된 작업   | 구조 변경 · 설계       |

---

### ⚡ 실험형 (Hacker)

빠르게 만들고 바로 시도해보는 스타일. 야간에 특히 활발하고 다양한 실험을 즐깁니다.

| 지표      | 신호              |
| --------- | ----------------- |
| 야간 비율 | 30% 이상          |
| 브랜치 수 | 많음 (실험용)     |
| 커밋 속도 | 빠름              |
| 주된 작업 | 프로토타입 · 실험 |

---

### 📚 탐구형 (Researcher)

깊이 있게 탐구하고 분석하는 개발 스타일. 데이터 분석 코드나 실험 노트북이 많습니다.

| 지표      | 신호               |
| --------- | ------------------ |
| 주요 언어 | Python / Jupyter   |
| repo 성격 | 분석 · 실험 코드   |
| 문서화    | README 상세        |
| 주된 작업 | 탐구 · 데이터 분석 |

---

### 🔧 장인형 (Craftsman)

디테일과 완성도를 중요하게 여깁니다. 작은 수정을 반복하고 테스트와 문서화에 공을 들입니다.

| 지표        | 신호                 |
| ----------- | -------------------- |
| 커밋 크기   | 매우 작음 (< 20라인) |
| 메시지 패턴 | fix / chore / docs   |
| 테스트 파일 | 비율 높음            |
| 주된 작업   | 정제 · 품질 개선     |

---

### 🧭 탐험형 (Explorer)

새로운 기술을 끊임없이 시도합니다. 여러 언어와 다양한 프로젝트를 넘나드는 것이 특징입니다.

| 지표        | 신호         |
| ----------- | ------------ |
| 사용 언어   | 5개 이상     |
| fork 비율   | 높음         |
| 프로젝트 수 | 다수 · 다양  |
| 주된 작업   | 새 기술 탐색 |

---

### 🛠️ 빌더형 (Builder)

무언가를 꾸준히 만들어내는 실행형 스타일. 자체 repo가 많고 지속적으로 개발을 이어갑니다.

| 지표        | 신호             |
| ----------- | ---------------- |
| 자체 repo   | 10개 이상        |
| 활동 지속성 | 장기 지속        |
| 커밋 메시지 | feat / add 중심  |
| 주된 작업   | 기능 개발 · 실용 |

## 4. 분석 로직

### 스코어링 방식

각 지표를 0~100으로 정규화하고, 유형별 가중치를 곱해 합산합니다.  
가장 높은 점수의 유형이 메인 결과로 출력됩니다.  
점수 차가 10% 이내면 "혼합형" 표기를 추가합니다 (예: 꾸준형 + 장인형).

### 주요 지표 정의

| 지표        | 측정 방법                                        |
| ----------- | ------------------------------------------------ |
| 활동 연속성 | 최근 30일 중 커밋이 있는 날 수                   |
| 집중도      | 단일 날짜 최대 커밋 수                           |
| 커밋 크기   | 커밋당 평균 변경 라인 수 (additions + deletions) |
| 야간 비율   | 22시~04시 사이 커밋 수 / 전체 커밋 수            |
| 언어 다양성 | repo 전반에서 사용된 언어 종류 수                |

### 꾸준형 판별 예시

```
활동일 > 20일        +30점
평균 커밋 크기 < 50라인  +25점
주말 활동 있음        +15점
야간 비율 < 20%      +10점
```

## 5. 결과 출력 구조

### 화면 구성

```
┌─────────────────────────────────┐
│  꾸준형 🌱 Gardener              │
│  72% 확률로 이 유형에 해당합니다  │
├─────────────────────────────────┤
│  작은 변화를 꾸준히 쌓아가는     │
│  개발 스타일입니다. ...          │
├─────────────────────────────────┤
│  AI 분석 근거                   │
│  · 최근 30일 중 22일 활동        │
│  · 평균 커밋 크기 38라인          │
│  · 유지보수 커밋 비율 64%         │
│  · 야간 커밋 비율 12%            │
└─────────────────────────────────┘
```

### README 배지

```markdown
![dev personality](https://dev-personality.com/badge/username)
```

출력 예시: `꾸준형 🌱 | Gardener`

### SNS 공유 카드

OG 이미지로 자동 생성. 유형명 + 이모지 + 한 줄 설명 포함. 트위터/링크드인 공유 최적화.

## 6. 사용자 경험 흐름

```
GitHub 로그인 (OAuth)
    ↓
분석 기간 선택 (30 / 90 / 180일)
    ↓
AI 분석 (3~5초)
  "커밋 패턴 분석 중..."
  "야간 활동 비율 계산 중..."
  "성향 도출 중..."
    ↓
결과 출력 (유형 + 근거)
    ↓
공유 (배지 / SNS 카드 / URL)
```

## 7. 차별화 포인트

| 포인트            | 설명                                                        |
| ----------------- | ----------------------------------------------------------- |
| 🎯 근거 기반 결과 | 결과에 AI 분석 근거 3~4개 표시 → "왜 이 유형인지" 납득 가능 |
| 📊 확률 표현      | "72% 확률로 꾸준형" → 단정 대신 추정치로 거부감 최소화      |
| 🔖 README 배지    | shields.io 스타일 배지로 GitHub 프로필에 삽입 → 자연 바이럴 |
| 🌐 한/영 병기     | "꾸준형 🌱 Gardener" — 국내외 모두 공유 가능한 이름 체계    |
